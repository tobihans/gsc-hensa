Gestion des commandes (ajout, modification, suppression)

Schéma de données relationnel (MySQL ou tout SGBD relationnel)
1. Table users
Champ	Type	Contraintes
id	INT	PK, AUTO_INCREMENT
name	VARCHAR(100)	NOT NULL
email	VARCHAR(150)	UNIQUE, NOT NULL
password	VARCHAR(255)	NOT NULL (hashé)
created_at	TIMESTAMP	DEFAULT CURRENT_TIMESTAMP
updated_at	TIMESTAMP	NULL

2. Table clients
Champ	Type	Contraintes
id	INT	PK, AUTO_INCREMENT
nom	VARCHAR(100)	NOT NULL
prenom	VARCHAR(100)	NOT NULL
email	VARCHAR(150)	UNIQUE, NULLABLE
telephone	VARCHAR(20)	NULLABLE
adresse	TEXT	NULLABLE
created_at	TIMESTAMP	DEFAULT CURRENT_TIMESTAMP
updated_at	TIMESTAMP	NULL

3. Table commandes
Champ	Type	Contraintes
id	INT	PK, AUTO_INCREMENT
client_id	INT	FK → clients(id), NOT NULL
user_id	INT	FK → users(id), NOT NULL
date_commande	DATETIME	DEFAULT CURRENT_TIMESTAMP
statut	ENUM	('en attente', 'en cours', 'livrée', 'annulée')
montant_total	DECIMAL(10,2)	NOT NULL
notes	TEXT	NULLABLE
created_at	TIMESTAMP	DEFAULT CURRENT_TIMESTAMP
updated_at	TIMESTAMP	NULL

Relations
Un utilisateur (user) peut enregistrer plusieurs commandes.

Une commande est associée à un seul client.

Un client peut avoir plusieurs commandes.
